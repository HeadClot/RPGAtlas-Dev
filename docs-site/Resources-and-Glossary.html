Resources & Glossary

A plain-language glossary, a few concept primers, and pointers for going further.

Glossary

Actor — An individual hero the player controls. Defined in the Database.

Auto Layer — The smart painting mode that sorts tiles onto the correct layer for you.

Autorun — An event trigger that runs by itself and locks player control (used for cutscenes).

Battler — The in-battle representation of a character or enemy.

Class — A template of stats, growth, traits, skills, and equipment permissions behind an actor.

Event — A scripted thing on a map: an NPC, a chest, a door, a cutscene trigger. See Events.

Encounter — A battle. Can be random (walking around) or fixed (a Start Battle command).

Export / Standalone build — A single .exe or .html file containing your whole game, for players. See Publishing Your Game.

Faceset / Face — A portrait shown beside dialogue.

HD-2D — An optional tilted-perspective renderer with extruded heights, lighting, and effects. See Maps & Tiles.

Layer — One of four stacked tile planes (Ground, Decor, Decor 2, Overhead).

localStorage — The browser's small storage box where the editor auto-saves. Tied to one browser; not a substitute for .json backups.

localhost — "This computer." Serving the engine at http://localhost means it runs entirely on your machine; nothing goes online.

Overhead — The top tile layer, drawn above the player so they pass behind it.

Page — One configuration of an event (graphic + trigger + conditions + commands). The last page whose conditions are met is the active one.

Passability — Whether a tile can be walked on (○) or blocks the player (✕).

Plugin — Project-embedded JavaScript that extends the engine. See Plugins.

Procedural — Generated by code at runtime rather than loaded from a file. RPGAtlas's tiles, sprites, enemies, sound, and music are all procedural.

Scope — Who a skill or item affects (one enemy, all enemies, one ally, the whole party…).

Self-Switch — An on/off flag local to a single event (A–D). The key to "do this once" logic.

Side view / Front view — The two battle presentations. See Battles & States.

State — A status effect (poison, stun, regen…). See Battles & States.

Switch — A game-wide on/off flag your events read and write.

Tile — One square of map art. Tiles are painted onto layers.

Troop — A group of enemies that appears together in one battle.

Trigger — What starts an event: Action Button, Player Touch, Autorun, or Parallel.

Variable — A game-wide number your events read and write.

Concept primers

What is an "RPG maker"?

A tool for building tile-based, story-driven role-playing games without writing an engine yourself. You assemble maps, characters, dialogue, and battles from building blocks and a visual editor. RPGAtlas is an original, free, open-source one.

The edit → playtest → repeat loop

Game-making is iterative. Make a small change, hit ▶ Playtest, see how it feels, adjust. Don't try to build everything before testing — short loops catch problems early and keep it fun.

Switches, variables, and self-switches (the mental model)

  • A switch is a light bulb for the whole game: on or off.
  • A variable is a counter for the whole game: a number.
  • A self-switch is a light bulb that belongs to one event only.

Most "this happened" logic is just flipping one of these and checking it later. See Events.

Licensing in plain language

  • The engine is licensed GPL-3.0-or-later. You can use, study, share, and modify it. If you distribute a modified engine, you share those engine changes under the same license.
  • Your game's content — maps, story, database entries, characters, custom art — is yours. It is not covered by the engine's license. Sell your games, no credit required.
  • Exports bundle the engine runtime (still GPL), but since exports are plain, readable HTML/JS, the source-availability requirement is satisfied by the export itself.

The full text is in the project's LICENSE file. (This page is a friendly summary, not legal advice.)

Going further

  • Read the built-in plugins (Plugin Manager) to learn the engine API by example.
  • Study the sample game, _Atlas Quest_, that ships with a fresh editor — open its maps and events to see how a complete little game is wired together.
  • The project README.md and the per-folder img/README.md cover formats and lower-level detail.
  • Pick a tiny scope for your first real project — one town, one dungeon, one boss. Finishing a small game teaches more than starting a huge one.

Where to get help

  • Re-check Troubleshooting & FAQ — most issues are covered there.
  • For bugs or questions about the engine itself, use the project's issue tracker / community channels on its hosting page.
